# 瑕果智选苹果分级规则 v0.1

更新时间：2026-05-28

## 1. 适用范围

本规则只适用于当前 MVP 的苹果单品类，用于 AI 初判、运营复核和消费者透明说明，不等同于正式食品安全检测标准。

## 2. 分级表

| 等级 | 典型标签 | 业务含义 | 食用边界 | 默认动作 | 建议流向 |
| --- | --- | --- | --- | --- | --- |
| A | `fresh` | 无明显瑕疵，外观完整度较高 | 不影响食用 | 可确认上架 | C 端鲜食 |
| B | `scab_defect` | 果锈、疮痂斑等表皮瑕疵 | 通常不影响果肉食用 | 可确认上架 | C 端性价比 |
| C | `bruise_defect` | 局部碰伤、需尽快处理 | 建议尽快食用 | 必须人工复核 | 复核后果切、榨汁或加工意向 |
| 禁售 | `rot_defect` | 疑似腐烂、霉变、破皮渗液 | 存在食用安全风险 | 强制拦截 | 禁售/剔除 |

## 3. 复核规则

以下情况必须进入人工复核：

- AI 识别为 `bruise_defect`。
- AI 识别为 `rot_defect`。
- `confidence < 0.7`。
- 图片不清晰、过曝、遮挡或无法判断瑕疵边界。
- 消费者反馈描述与商品说明不一致。

## 4. 消费者文案边界

允许表达：

- “基于图片初步判断”
- “通常不影响果肉食用”
- “建议收到后优先食用”
- “如与描述不符可反馈复盘”

禁止表达：

- “绝对安全”
- “保证无风险”
- “完全没问题”
- “百分百安全”
- “一定安全”

## 5. 当前验收标准

| 指标 | MVP 合格线 |
| --- | --- |
| 腐烂样本漏放率 | 0% |
| 高风险召回率 | 100% |
| JSON 可解析率 | >= 95% |
| 等级一致率 | >= 80% |
| 文案合规率 | 100% |

## 6. 后续扩展

下一版规则应补充：

- 瑕疵面积比例。
- 是否破皮。
- 是否软烂。
- 是否疑似霉变。
- 是否需要补图。
- 采摘时间和运输时效对建议流向的影响。
