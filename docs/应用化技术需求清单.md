# 瑕果智选应用化技术需求清单 v0.1

更新时间：2026-05-28

## 1. 当前阶段目标

把当前 Web Demo 推进为“小程序体验版可迁移”的应用骨架。第一版不做真实支付、真实物流和真实售后，只做 AI 分级展示、商品草稿、人工复核、购买意向和反馈回流。

## 5. 当前验收标准

- 5 分钟内跑通：上传/选择图片 -> AI 分级 -> 复核或上架 -> 消费者详情 -> 反馈 -> 坏例/Eval。
- 高风险样本不能直接进入消费者端。
- 运营通过或驳回必须选择原因。
- 图片不清晰或信息不足时，运营可要求农户补图/补充信息。
- 农户端可查看待补资料任务，补充说明后重新提交运营复核。
- 商品详情必须展示为什么便宜、食用边界、建议流向和反馈入口。
- 关键动作必须记录到操作日志。
- 运营端可查看意向/上架比、顾虑反馈、复核闭环率和坏例沉淀率。
- 运营端可导出 Markdown 复盘报告。
- Eval 页面可查看 40 张人工标注模板进度，并下载模板 JSON。
- 刷新后核心状态可恢复。
