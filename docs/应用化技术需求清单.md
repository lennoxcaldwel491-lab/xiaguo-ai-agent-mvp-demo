## 5. 小程序数据同步方案

当前小程序新增统一数据访问层：

```text
miniprogram/utils/data-store.js
```

当前策略：

- 优先请求本地 API：`GET /api/state`、`POST /api/state`。
- API 不可用时回退 `wxStorage`。
- 服务端和本地都有数据时按 `id` 合并。
- 目前同步对象包括：
  - `products`
  - `feedbacks`
  - `badCases`
  - `evalRuns`

注意：

- 小程序体验版当前默认 `apiBase` 为 `http://127.0.0.1:8787`。
- 微信开发者工具或真机调试时，`127.0.0.1` 不一定指向电脑本地服务。
- 正式体验版需要替换为局域网 IP、内网穿透 HTTPS 地址或正式 HTTPS 域名。

## 6. 当前验收标准

- 5 分钟内跑通：上传/选择图片 -> AI 分级 -> 复核或上架 -> 消费者详情 -> 反馈 -> 坏例/Eval。
- 高风险样本不能直接进入消费者端。
- 运营通过或驳回必须选择原因。
- 图片不清晰或信息不足时，运营可要求农户补图/补充信息。
- 农户端可查看待补资料任务，补充说明后重新提交运营复核。
- 商品详情必须展示为什么便宜、食用边界、建议流向和反馈入口。
- 关键动作必须记录到操作日志。
- 运营端可查看意向/上架比、顾虑反馈、复核闭环率和坏例沉淀率。
- 运营端可导出 Markdown 复盘报告。
- Eval 页面可查看 40 张人工标注模板进度，并下载模板 JSON。
- 刷新后核心状态可恢复。
- 小程序体验版可跑通：农户上传 -> 填写基础信息 -> 商品状态 -> 消费者详情 -> 反馈 -> 运营复核 -> 坏例/Eval。
