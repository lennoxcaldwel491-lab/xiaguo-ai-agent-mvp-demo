# 瑕果智选 AI Agent 下一阶段路径图

## 当前状态

第一阶段 MVP 已经完成：农户上架、AI 分级、人工复核、消费者商品页、反馈回流、坏例池和 Eval 页面都已跑通。

当前已经接入 Qwen-VL / 阿里百炼视觉模型，API 模式可返回真实模型识别结果。项目从“mock 产品闭环”进入“真实 AI 可靠性验证”阶段。

## 阶段 2：真实 AI 冒烟测试

目标：确认真实模型能稳定看图、返回固定 JSON，并且不会绕过食品安全规则。

要做：

- 分别测试 4 类苹果样本：无明显瑕疵、果锈/疮痂斑、轻微碰伤、疑似腐烂。
- 每类至少测 3 张，共 12 张。
- 记录模型输出的 `defect_type`、`grade`、`confidence`、`review_required`、`risk_flags`。
- 重点检查：腐烂图必须进入人工复核或禁售，不能直接上架。

验收标准：

- API 模式能连续跑完 12 张样本。
- 返回字段结构稳定，不影响前端展示。
- 疑似腐烂样本 100% 进入人工复核。
- 低置信度结果 100% 进入人工复核。

## 阶段 3：建立人工标注表

目标：把图片数据集从“目录标签”升级为“业务可用标注集”。

要做：

- 从苹果数据集中先抽 40 张图：每类 10 张。
- 为每张图补充人工标注字段：
  - 水果品类
  - 数据集标签
  - 业务瑕疵类型
  - 人工建议等级
  - 是否影响食用安全
  - 是否必须复核
  - 备注
- 把标注结果保存为 `eval/apple_eval_set.json`。

验收标准：

- 至少 40 张图有人工标注。
- 每个样本都能和 AI 输出做对比。
- 能计算腐烂漏判、可售果误拦截、等级一致率。

## 阶段 4：真实 AI Eval

目标：用真实图片评估模型是否能承担 MVP 初判任务。

核心指标：

- 腐烂风险漏判率：疑似腐烂被判为可上架的比例，目标为 0。
- 强制复核召回率：高风险样本进入复核的比例，目标为 100%。
- 等级一致率：AI 等级与人工等级一致的比例，先以 70% 为 MVP 参考线。
- 可售果误拦截率：A/B 级样本被误判为禁售或复核的比例，先观察不强求。
- JSON 稳定率：返回结构可被前端解析的比例，目标为 100%。

验收标准：

- 能输出一份 Eval 报告。
- 能列出失败样本和失败原因。
- 能把失败样本进入坏例池。

## 阶段 5：优化 Agent 提示词和规则

目标：根据坏例修正模型行为，而不是盲目换模型。

要做：

- 把腐烂、霉变、破皮渗液作为强规则写入提示词。
- 要求模型区分“外观瑕疵”和“食品安全风险”。
- 要求模型在看不清时输出低置信度，并进入人工复核。
- 为消费者文案加限制：不能承诺绝对安全，只能说“基于图片初步判断”。

验收标准：

- 坏例复测后不再出现同类高风险漏判。
- 消费者文案不过度承诺。
- 农户解释能说明为什么需要复核。

## 阶段 6：补齐产品化能力

目标：让 MVP 更接近可交付演示版本。

要做：

- 增加 Eval 结果导出。
- 增加真实 AI 输出与人工标注对比表。
- 增加复核原因必填。
- 增加坏例类型筛选。
- 增加“重新运行 AI 分级”按钮。
- 增加商品状态流转日志。

验收标准：

- 能完整演示一次：AI 误判 -> 人工修正 -> 进入坏例 -> 更新规则 -> 复测。
- 项目说明里能讲清楚 AI 能力边界和人工责任边界。

## 阶段 7：第二品类扩展

目标：在苹果跑通后，再考虑芒果。

不要马上做多品类。苹果阶段稳定后，再加入芒果图片和规则。

扩展前提：

- 苹果 40 张 Eval 跑通。
- 腐烂风险漏判率为 0。
- 坏例池和规则迭代流程稳定。

## 最近 3 天建议

第 1 天：
跑 12 张真实 API 样本，记录明显错误和不稳定输出。

第 2 天：
建立 40 张苹果人工标注集，先不用全量数据库。

第 3 天：
做真实 AI Eval 报告，确定是否需要调提示词或更换模型。

## 一句话结论

当前 MVP 已经完成“产品闭环”，下一阶段的核心不是继续加页面，而是验证真实视觉模型的可靠性：尤其是食品安全风险能不能被稳定拦截。
