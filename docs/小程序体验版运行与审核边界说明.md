# 瑕果智选小程序体验版运行与审核边界说明

更新时间：2026-06-01

## 1. 当前版本定位

当前 `miniprogram/` 是瑕果智选小程序体验版骨架，用于演示苹果瑕疵果 AI 辅助分级、人工复核、可信商品说明、购买意向收集、坏例池和 Eval 回归检查。

当前版本不是正式交易小程序，不提供真实支付、真实物流、真实售后履约和真实订单服务。

## 2. 本地运行前提

### Web/后端服务

在项目根目录启动本地服务：

```powershell
cd "C:\Users\PC\Desktop\全知全能\项目\瑕疵果\瑕果智选_AI_Agent_MVP_苹果版"
node server.js
```

本地服务默认地址：

```text
http://127.0.0.1:8787
```

可检查：

```text
http://127.0.0.1:8787/api/health
```

### 小程序项目

用微信开发者工具打开：

```text
miniprogram/
```

当前 `project.config.json` 使用：

```json
{
  "appid": "touristappid",
  "projectname": "xiaguo-zhixuan-experience"
}
```

这是体验开发配置，不等同于正式小程序 AppID。

## 3. API 地址说明

小程序统一数据访问层：

```text
miniprogram/utils/data-store.js
```

当前默认 API 地址在：

```text
miniprogram/app.js
```

```js
apiBase: "http://127.0.0.1:8787"
```

调试注意：

- 在微信开发者工具模拟器里，`127.0.0.1` 通常指向开发机本机。
- 在真机调试时，`127.0.0.1` 指向手机自身，不会访问电脑上的 Node 服务。
- 真机调试需要把 `apiBase` 替换为电脑局域网 IP，例如 `http://192.168.x.x:8787`。
- 正式体验版/审核版需要 HTTPS 域名，并在微信公众平台配置 request 合法域名。

当前 `data-store` 策略：

- 优先同步本地 API `/api/state`。
- API 不可用时回退 `wxStorage`。
- 服务端和本地都有数据时按 `id` 合并，避免演示数据被覆盖。

## 5. 演示路径

### 农户路径

```text
首页 -> 农户端 -> 拍摄/选择苹果 -> 填写基础信息 -> 开始智能分级 -> 查看单果状态
```

状态包括：

- 已上架
- 复核
- 已售出
- 禁售
- 待补充

### 消费者路径

```text
首页 -> 消费者端 -> 商品列表 -> 商品详情 -> 提交购买意向/顾虑反馈
```

消费者端只展示已上架商品。

### 运营路径

```text
首页 -> 运营端 -> 复核队列 -> 通过上架/要求补资料/禁售并记坏例
```

运营禁售、AI 直接禁售和消费者负向反馈会进入坏例池。

### Eval 路径

```text
首页 -> Eval -> 查看坏例池和指标 -> 运行本地回归检查
```

## 6. 审核边界声明

体验版对外说明建议使用：

> 本小程序用于苹果瑕疵果 AI 辅助分级展示与购买意向调研。AI 分级结果仅作为初步参考，不构成最终食品安全检测结论；疑似腐烂、霉变、破皮渗液、低置信度或消费者反馈异常的样本需人工复核。当前版本不提供在线支付、真实物流发货和正式售后履约。

## 8. 不能在体验版中承诺的内容

- 不能承诺 AI 判断绝对准确。
- 不能承诺“绝对安全”“保证无风险”。
- 不能暗示已完成真实食品安全检测。
- 不能展示真实支付、真实发货或真实售后能力。
- 不能把购买意向描述成真实订单。
- 不能在没有资质和履约能力前宣传正式电商交易。
